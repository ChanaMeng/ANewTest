# 商业化与技术方案（微信小游戏）

> **目标平台**: 微信小游戏 (竖屏 9:16)
> **商业模式**: 免费下载 + 激励广告 + 内购(IAP) 混合变现
> **对标商业化**: 《叫我大掌柜》《咸鱼之王》《寻道大千》
> **文档版本**: V2.0

---

## 一、微信小游戏商业化总览

### 1.1 混合变现模型

微信小游戏用户付费习惯与原生手游存在显著差异：**小额高频付费为主，广告容忍度高**。因此本游戏采用"广告 + 内购"双轮驱动模型。

```
              ┌──────────────────────────────────┐
              │     搜打撤俱乐部 商业化架构        │
              ├──────────────────────────────────┤
              │                                  │
              │  ┌────────────┐  ┌────────────┐  │
              │  │  激励广告   │  │  内购 IAP   │  │
              │  │  (70%流量)  │  │ (30%流量)   │  │
              │  ├────────────┤  ├────────────┤  │
              │  │·护送加速    │  │·首充礼包    │  │
              │  │·双倍收益    │  │·月卡订阅    │  │
              │  │·免费抽卡    │  │·赛季通行证   │  │
              │  │·资源补充    │  │·直购暗金    │  │
              │  │·离线翻倍    │  │·限时礼包    │  │
              │  └────────────┘  └────────────┘  │
              │                                  │
              │  广告ARPU ≈ ¥0.3-0.8/DAU/日      │
              │  内购ARPU ≈ ¥2-5/付费用户/日      │
              │                                  │
              └──────────────────────────────────┘
```

### 1.2 用户付费分层

| 用户层级 | 占比 | 月均付费 | 主要付费点 | 贡献占比 |
|----------|------|----------|------------|----------|
| 零氪（广告白嫖） | 45% | ¥0 | 激励广告获取资源 | 0%（内购）/ 20%（广告收入） |
| 轻度付费 | 30% | ¥6-30 | 首充+月卡 | 15%（内购）+ 35%（广告收入） |
| 中度付费 | 18% | ¥68-200 | 月卡+通行证+礼包 | 35% |
| 重度付费 | 5% | ¥500-3000 | 满图鉴+直购+全通 | 35% |
| 大R | 2% | ¥3000-10000+ | 全满+外观收集 | 15% |

### 1.3 微信小游戏付费特点

| 特点 | 设计策略 |
|------|----------|
| **首充转化关键** | ¥6超值礼包，30分钟内弹窗引导，首充率目标 8-12% |
| **广告ARPU核心** | 每日广告点位饱满但不强制，目标广告填充率 > 60% |
| **月卡续费率** | 月卡到期前3天开始提醒，续费率目标 40-50% |
| **小额高频** | 单个付费点不超过¥128（除大R直购档），降低决策门槛 |
| **社交裂变** | 分享得招募券、微信群排行、助力领好礼 |
| **沉默唤醒** | 订阅消息推送：委托完成、至宝发现、猛攻节开启 |

## 二、激励广告点位设计（核心收入支柱）

激励广告是微信小游戏最重要的收入来源之一。设计原则：**高频场景 × 低决策成本 × 即时满足**。

### 2.1 广告点位全景

| 编号 | 场景 | 广告类型 | 每日上限 | 用户收益 | 预期点击率 | 设计目的 |
|------|------|----------|----------|----------|------------|----------|
| AD-01 | **护送任务加速** | 激励视频(30s) | 5次/日 | 跳过等待时间，立即结算 | 70% | 核心场景，频率最高 |
| AD-02 | **双倍直播收益** | 激励视频(30s) | 3次/日 | 当日直播室收益×2 | 60% | 放置收益放大器 |
| AD-03 | **紧急招募** | 激励视频(30s) | 2次/日 | 免费高级抽卡×1 | 80% | 抽卡欲望驱动 |
| AD-04 | **装备维护补充** | 激励视频(15-30s) | 3次/日 | 补充维护资源(装备零件×20) | 55% | 资源缺口驱动 |
| AD-05 | **离线收益翻倍** | 激励视频(30s) | 1次/日 | 离线挂机收入×2 | 85% | 上线首屏必弹 |
| AD-06 | **委托结算翻倍** | 激励视频(30s) | 3次/日 | 单次委托结算奖励×2 | 65% | 结算时即时诱惑 |
| AD-07 | **每日幸运转盘** | 激励视频(30s) | 1次/日 | 额外一次转盘机会 | 75% | 博彩心理 |
| AD-08 | **训练加速** | 激励视频(15s) | 2次/日 | 训练时间减半 | 40% | 低频补充 |
| AD-09 | **情报碎片** | 激励视频(15-30s) | 1次/日 | 额外情报碎片×3 | 50% | 系统资源补充 |
| AD-10 | **对抗赛复活** | 激励视频(30s) | 2次/周 | 对抗赛失败后可重试 | 60% | 竞技场景 |

**每日广告总上限**: 约 17-20 次（不含每周限次）
**设计eCPM预期**: 激励视频 ¥15-35/千次（微信小游戏行业水平）
**广告ARPU预期**: ¥0.3-0.8/DAU/日（按平均3-5次完成计算）

### 2.3 广告频控策略

| 维度 | 策略 |
|------|------|
| **全局上限** | 单用户每日最多观看 20 次激励广告，防止疲劳 |
| **场景上限** | 每个广告点位独立计数，不互相挤占 |
| **间隔保护** | 同一点位广告间隔 ≥ 5 分钟（离线翻倍除外） |
| **VIP豁免** | 购买月卡用户部分广告点位自动获得收益（如离线翻倍） |
| **广告降级** | 广告加载失败时给予50%收益作为补偿，保护体验 |

## 四、ARPU 估算模型

### 4.1 微信小游戏行业基准

| 指标 | 行业平均 | 行业优秀 | 本游戏目标 |
|------|----------|----------|------------|
| 次留 | 20-30% | 35-45% | 35% |
| 7留 | 8-12% | 15-20% | 15% |
| 30留 | 3-5% | 8-12% | 8% |
| 首充率 | 3-5% | 8-12% | 10% |
| 付费率 | 2-4% | 5-8% | 5% |
| 广告ARPU | ¥0.15-0.4 | ¥0.5-1.0 | ¥0.5 |
| 内购ARPPU | ¥30-80 | ¥100-200 | ¥120 |
| eCPM | ¥10-20 | ¥25-35 | ¥25 |

### 4.2 首月收入预估模型

假设首月 DAU = 50,000（通过买量+自然量+社交裂变）：

```
收入 = 广告收入 + 内购收入

广告收入：
  DAU 50,000 × 广告渗透率 60% × 日均3次 × eCPM ¥25/1000
  = 50,000 × 0.6 × 3 × 0.025
  = ¥2,250/日
  = ¥67,500/月

首月内购收入：
  付费用户数 = DAU 50,000 × 付费率 5% = 2,500人

  首充用户 = 50,000 × 首充率 10% = 5,000人 × ¥6 = ¥30,000
  月卡用户 = 2,500 × 40% = 1,000人 × ¥30 = ¥30,000
  豪华月卡 = 2,500 × 10% = 250人 × ¥98 = ¥24,500
  通行证(68) = 2,500 × 20% = 500人 × ¥68 = ¥34,000
  通行证(128) = 2,500 × 8% = 200人 × ¥128 = ¥25,600
  其他直购/礼包 = ¥50,000(估算)

  内购合计 ≈ ¥194,100/月

首月总收入 ≈ ¥261,600

首月ARPU = ¥261,600 / 50,000 ≈ ¥5.23
首月ARPPU = ¥194,100 / 2,500 ≈ ¥77.64
```

### 4.3 稳定期收入预估

假设稳定期 DAU = 30,000（自然衰减后）：

wx.requestPayment({
  timeStamp: '',
  nonceStr: '',
  package: '',   // 后端统一下单返回的 prepay_id
  signType: 'MD5', // 或 'HMAC-SHA256'
  paySign: '',
  success() { /* 发货逻辑 */ },
  fail() { /* 支付失败处理 */ }
})

// ===== 3. 激励视频广告 =====
const rewardedVideoAd = wx.createRewardedVideoAd({
  adUnitId: 'adunit-xxxxxxxx' // 每个广告位独立 adUnitId

// 预加载广告（提高展示成功率）

// 展示广告
rewardedVideoAd.show()
  .catch(() => {
    // 加载失败，重新加载

      .then(() => rewardedVideoAd.show())

rewardedVideoAd.onClose(res => {
  if (res && res.isEnded) {
    // 用户完整观看，发放奖励
  } else {
    // 用户中途关闭，不发放或发放部分
  }
})

// ===== 4. 订阅消息 =====
wx.requestSubscribeMessage({
  tmplIds: [
    'template_id_1', // 委托完成通知
    'template_id_2', // 猛攻节开启通知

### 6.8 支付回调防作弊

```
支付流程：
1. 客户端调用 wx.requestPayment 发起支付
2. 微信回调后端 notify_url
3. 后端验签 → 查询订单 → 发放道具 → 更新数据
4. 客户端轮询订单状态 / WebSocket推送

关键防作弊措施：
- 所有道具发放必须服务端验签后进行
- 客户端不可直接写入货币数据
- 广告奖励通过服务端 ad_unit_id + timestamp 防重放
- 关键战斗结果服务端验算（非纯客户端计算）
- 数据库双写（MySQL记录 + Redis缓存）防丢失
```

## 九、风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| **广告eCPM不及预期** | 中 | 高 | 增加内购付费点密度，提升付费转化率补偿 |
| **买量成本过高** | 高 | 中 | 强化社交裂变（分享得招募券），降低对买量依赖 |
| **内容消耗过快** | 中 | 中 | 确保50+随机事件池，赛季更新频率不低于12周 |
| **微信审核不通过** | 低 | 高 | 严格遵守内容合规，世界观架空，不做真实映射 |
| **小游戏包体限制** | 中 | 中 | Cocos首包<4MB，远程资源分包加载 |
| **付费玩家差评** | 低 | 中 | 保证零氪/微氪体验，付费只是加速非必须 |

## 十、附录

### 10.1 微信小游戏平台规则摘要

| 规则 | 说明 |
|------|------|
| **首包大小** | 不超过4MB（代码包），资源分包无限制 |
| **支付分成** | 微信渠道服：平台分成40%（普通），30%（创意小游戏） |
| **广告分成** | 流量主广告收入：平台分成30%-50%（按广告类型） |
| **虚拟支付** | 仅支持人民币，不支持虚拟币→人民币反向兑换 |
| **诱导分享** | 禁止"分享后解锁"等强制分享，但允许分享后双方获奖励 |
| **未成年人保护** | 必须接入微信未成年人防沉迷系统 |

### 10.2 广告分成模型计算

```
假设日广告收入 ¥2,250：
  平台分成 40% = ¥900
  开发者净收入 = ¥1,350/日

假设月内购收入 ¥194,100：
  平台分成 40% = ¥77,640
  开发者净收入 = ¥116,460/月

首月开发者净收入 ≈ ¥116,460 + ¥40,500 = ¥156,960
```

### 10.3 关键术语

| 术语 | 说明 |
|------|------|
| eCPM | Effective Cost Per Mille，每千次广告展示有效收入 |
| ARPU | Average Revenue Per User，每用户平均收入 |
| ARPPU | Average Revenue Per Paying User，每付费用户平均收入 |
| LTV | Lifetime Value，用户生命周期价值 |
| DAU | Daily Active Users，日活跃用户数 |
| K因子 | 病毒传播系数，衡量社交裂变效率 |
| 填充率 | 广告请求成功返回广告的比例 |
| 完成率 | 用户完整观看广告的比例 |
| CBT | Closed Beta Test，封闭测试 |
