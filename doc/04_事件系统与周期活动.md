# 事件系统与周期活动（V2 · 事件日志系统）

> **核心设计理念**：事件系统是《搜打撤俱乐部》的**核心叙事载体**。借鉴"人生模拟器（Life Simulator / BitLife）"的日志流体验，将抽象的资源变化转化为具象的、可阅读的、有情感张力的文字日志。玩家每天刷到的不是冰冷的数值表，而是一条条"俱乐部日常"的鲜活故事。

---

## 一、事件日志系统 · 核心设计

### 1.1 什么是事件日志系统

事件日志系统（Event Log System）是用**滚动时间线 + 日志条目**呈现游戏进程的 UI 框架。它取代传统事件弹窗，成为玩家感知"俱乐部在运转"的主要窗口。

```
┌──────────────────────────────┐
│  第23天    4月15日 晴         │
│  俱乐部资金：12,450 游戏币    │
│  人气值：★ ★ ★ ☆ ☆          │
│                              │
│  ── 今日事件 ─────────────── │
│  🟢「刮地皮在护送中意外发现   │
│      隐藏物资！+300游戏币     │
│      [人气+5]」              │
│                              │
│  🟢「直播间礼物特效炸裂，     │
│      老板一掷千金！+200游戏币 │
│      [人气+3]」              │
│                              │
│  🔴「设备故障！直播中断30分钟 │
│      -100游戏币[人气-2]」    │
│                              │
│  🟡「神秘委托人深夜来电，     │
│      悬赏一件特殊任务…       │
│      [查看详情 →]」          │
│                              │
│  [一键收取奖励]              │
└──────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **具象化** | 每条日志包含具体角色名、具体数值变化、具体场景 |
| **故事感** | 好的事件让玩家脑补出一段小剧情，差的只是数字播报 |
| **有代价** | 坏事不是"概率生效"，而是"正在发生"的叙事感 |
| **可期待** | 玩家每天登录的动力 = "今天会发生什么？" |
| **非打扰** | 不弹窗阻塞操作，日志滚动呈现，可一键收取 |

### 1.3 每日事件节奏

| 时段 | 触发事件数量 | 事件类型 |
|------|-------------|----------|
| **晨间刷新**（5:00） | 2-4 条 | 俱乐部事件 + 外部事件 |
| **护送期间**（全天） | 2-4 条 | 护送事件（每次委托完成后触发 1-2 条） |
| **直播收益**（20:00） | 1-3 条 | 直播事件（全天直播活动的总结） |
| **夜间接力**（离线） | 1-2 条 | 离线事件（下次登录时一次性展示） |

**每日合计：5-8 条事件**，高峰期（运营活动/猛攻节）可增至 8-12 条。

## 二、事件分类与完整事件池

### 2.1 事件分类总览

| 分类 | 颜色 | 触发时机 | 典型内容 |
|------|------|----------|----------|
| **护送事件** | 🟢🟡🔴 | 委托完成后 | 搜刮奇遇、战斗高光、翻车惨案、撤离意外 |
| **直播事件** | 🟢🟡🔴 | 每日结算时 | 爆火/事故、打赏/流失、平台互动 |
| **俱乐部事件** | 🟢🟡🔴 | 晨间刷新 + 随机 | 员工动态、设施状况、内部纠纷 |
| **外部事件** | 🟢🟡🔴 | 晨间刷新 + 运营 | 行业新闻、竞争动态、神秘来客 |

**颜色编码**：
- 🟢 **绿色** — 好事（增益事件）
- 🔴 **红色** — 坏事（损失事件）
- 🟡 **金色** — 奇遇（稀有事件，影响重大）

## 十、事件生成算法

### 10.1 事件权重配比

每种事件类型的每日触发概率：

| 事件类型 | 日常权重 | 猛攻节权重 |
|----------|---------|-----------|
| 护送好事 | 35% | 40% |
| 护送坏事 | 20% | 20% |
| 护送奇遇 | 5% | 15% |
| 直播好事 | 12% | 20% |
| 直播坏事 | 10% | 8% |
| 直播奇遇 | 3% | 12% |
| 俱乐部好事 | 6% | 8% |
| 俱乐部坏事 | 6% | 5% |
| 俱乐部奇遇 | 1% | 3% |
| 外部事件 | 2% | 5% |

### 10.2 价值控制公式

```
单日净产出 ≈ 事件增益总和 - 事件损失总和

目标区间：
- 普通日：+200 ~ +800 游戏币净收益
- 幸运日：+800 ~ +1500 游戏币净收益
- 倒霉日：-200 ~ +200 游戏币净收益（罕见）
- 猛攻节第1天：+1000 ~ +2000 游戏币净收益
- 猛攻节第3天：+1500 ~ +3000 游戏币净收益（保底不亏）

保护机制：
- 如果前一天是净损失日，次日好事权重+15%
- 如果连续3天净损失，强制触发补偿性奇遇事件
- 新手前7天：屏蔽C11-C15级坏事件（保证新手体验友好）
```

### 10.3 事件防重复

```
规则：
1. 同一事件24小时内不会重复触发
2. 同类型事件（如两个不同的事件链）不会在同一天开启
3. 事件链活跃期间，保护该链不被其他事件链中断
4. 事件池按玩家进度动态解锁：
   - Lv1~5：  基础事件池（E01-E08, L01-L04, C01-C06）
   - Lv6~10： 扩展事件池（E09-E16, L05-L10, C07-C12）
   - Lv11+：  完整事件池（E17-E24, L11-L17, C13-C19, X全部, 奇遇全部）
```
