# 社交与排行榜系统

## 一、微信小游戏社交特性

### 1.1 社交功能总览

微信小游戏的天然社交优势是本作的竞争力核心。基于微信关系链，构建以下社交体系：

| 功能模块 | 描述 | 解锁条件 | 涉及系统 |
|----------|------|----------|----------|
| 好友系统 | 微信好友即为游戏好友 | 初始 | 全部 |
| 好友俱乐部访问 | 参观好友基地 | 基地Lv3 | 社交、养成 |
| 员工借调 | 临时借用好友员工 | 好友Lv3友情等级 | 委托、社交 |
| 排行榜 | 多维度排名 | 基地Lv5 | 竞技、社交 |
| 俱乐部联盟 | 公会系统 | 基地Lv15 | 社交、活动 |
| 对抗赛 | 俱乐部PVP | 基地Lv10 | 竞技、排行 |
| 分享机制 | 分享战果到群聊 | 初始 | 裂变增长 |
| 订阅消息 | 定时提醒推送 | 用户授权后 | 留存 |

### 1.2 微信关系链导入

```
游戏好友 = 微信好友 + 游戏内ID搜索添加

好友关系自动同步规则：
  - 微信好友自动成为游戏好友（需双方授权）
  - 群聊成员可"一键添加"为游戏好友
  - 不暴露微信昵称外的任何隐私信息
  - 可设置"隐身模式"（对特定好友不可见）
```

## 五、俱乐部对抗赛（PVP）

### 5.1 对抗赛概述

俱乐部对抗赛是核心PVP玩法，采用异步战斗模式（无需实时在线）。

| 项目 | 设定 |
|------|------|
| 解锁条件 | 作战指挥室Lv10 |
| 开放时间 | 每天12:00-14:00 / 20:00-22:00 |
| 次数限制 | 免费5次/天，可购买额外次数（暗金10/次，上限10次） |
| 匹配机制 | 基于战力分段匹配（±10%战力） |
| 战斗模式 | 3v3 员工对阵，自动战斗 |

### 5.2 对抗赛流程

```
┌─────────────────────────────────┐
│ 1. 匹配对手                     │
│    ├── 战力分段匹配              │
│    ├── 显示对手信息（隐藏阵容）   │
│    └── 确认进入                  │
├─────────────────────────────────┤
│ 2. 阵容编排（60秒）              │
│    ├── 选择3名上场员工           │
│    ├── 配置战术指令              │
│    └── 确认阵容                  │
├─────────────────────────────────┤
│ 3. 战斗结算（自动）              │
│    ├── 战斗日志展示（可跳过）     │
│    ├── 胜/负/平判定              │
│    └── 奖励结算                  │
├─────────────────────────────────┤
│ 4. 战后回顾                     │
│    ├── 战斗数据面板              │
│    ├── 分享战报到群聊            │
│    └── 复仇（再战）              │
└─────────────────────────────────┘
```

### 5.3 对抗赛积分与段位

| 段位 | 积分范围 | 段位奖励（赛季结算） | 图标 |
|------|----------|---------------------|------|
| 青铜 | 0-999 | 信用点 10,000 | 🟤 |

| 钻石 | 4,000-4,999 | 暗金 100 + 限定头像框 | 💎 |
| 大师 | 5,000-5,999 | 暗金 200 + 专属皮肤 | 🟣 |

## 七、订阅消息与推送策略

### 7.1 订阅消息模板

微信小游戏订阅消息限制严格，需精心设计触发时机：

| 模板名称 | 触发时机 | 消息内容 | 合规要点 |
|----------|----------|----------|----------|
| 设施完成通知 | 设施升级完成 | "作战指挥室已升级到Lv{X}" | 操作结果通知 |
| 委托完成通知 | 护送委托完成 | "护送任务已完成，获得{X}俱乐部资金" | 服务通知 |
| 体力恢复通知 | 体力补满 | "体力已恢复，快回来继续经营吧" | 状态通知 |
| 赛季结算通知 | 赛季结束前24h | "赛季即将结束，当前排名第{X}" | 赛事通知 |
| 对抗赛提醒 | 对抗赛开启前30分钟 | "对抗赛即将开始，记得来参加" | 活动提醒 |
| 好友互动通知 | 好友参观/借调 | "{好友名}参观了你的俱乐部" | 社交通知 |

### 7.2 推送策略

| 用户类型 | 推送频率 | 推送时段 | 内容策略 |
|----------|----------|----------|----------|
| 活跃用户 | 2-3条/天 | 10:00-22:00 | 限时活动、对抗赛提醒 |
| 中度用户 | 1-2条/天 | 12:00/20:00 | 体力恢复、委托完成 |
| 轻度用户 | 1条/天 | 20:00 | 每日奖励提醒 |
| 沉默用户（3天未登录） | 暂停推送 | - | 仅好友互动被动通知 |
| 流失用户（7天未登录） | 1条/3天 | 20:00 | 回流奖励通知 |

### 7.3 推送文案设计原则

```
推送文案设计原则：
1. 避免机械通知，用游戏内角色口吻
   ❌ "你的设施升级完成"
   ✓ "「锁子哥」报告：基地设施升级完毕！快回来看看变化~"

2. 制造紧迫感但不制造焦虑
   ✓ "猛攻节今晚24点结束，你的积分还差200就能领大奖了！"
   ❌ "再不上线你的排名就要掉了！"

3. 突出奖励和成长
   ✓ "离线收入已累计：游戏币 58,000，一键收取！"
```
